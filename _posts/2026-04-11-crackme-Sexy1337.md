## Challenge

A simple password checker binary. The goal is to find the correct password.

## Approach

**Step 1 — Initial recon**

First, make the binary executable and run it to see what we're dealing with. Inputting a few random characters gives us `"Access Denied!"`

![fail](/assets/images/2026-04-11/fail.png)

**Step 2 — Opening Ghidra**

Loading the binary into Ghidra and checking the functions list, a few stand out `main`, `generate_password` and `encrypt_decrypt`.

![functions](/assets/images/2026-04-11/function-tree.png)

**Step 3 — Reversing main**

Decompiling `main` reveals the following logic:

1. calls generate_password(local_48, "sexy1337")
2. calls encrypt_decrypt(local_48,0xffffffaa)
3. prompts input via `fgets` stores in local_88
4.  calls encrypt_decrypt(local_88,0xffffffaa); local_ 88 = input
5. compares the generated password stored in local_48 to your input stored in local_88 via `strcmp`

![main](/assets/images/2026-04-11/main.png)


**Step 4 — Reversing generate_password**

This function just takes the string `sexy1337` and each byte is shifted by 3 like this:

| Char | ASCII | Shift | Result     |
|:----:|:-----:|:-----:|------------|
| s    | 0x73  | +3    | v (0x76)   |
| e    | 0x65  | +3    | h (0x68)   |
| x    | 0x78  | +3    | { (0x7B)   |
| y    | 0x79  | +3    | \| (0x7C)  |
| 1    | 0x31  | +3    | 4 (0x34)   |
| 3    | 0x33  | +3    | 6 (0x36)   |
| 3    | 0x33  | +3    | 6 (0x36)   |
| 7    | 0x37  | +3    | : (0x3A)   |

so the result is `"vh{|466:"`

![passgen](/assets/images/2026-04-11/generate-password.png)


**Step 5 —  Investigating encrypt_decrypt**

Although we don’t actually need to use this function to solve the challenge, it’s worth mentioning. The `encrypt_decrypt` function simply XORs each byte of a string with a single-byte key. Since both the generated password and the user input are XORed with the same key, the transformations are identical, so the XOR effectively **cancels out** during the comparison.

![xor](/assets/images/2026-04-11/encrypt-decrypt.png)

**Step 6 —  Writing a Python script**

Since we only need to reverse generate_password which is just shifting the string: `sexy1337` to get the password we can write a small Python script that shifts each byte by 3 for us like this:
```python
string = "sexy1337"

# Shift each character by 3
shifted = []
for char in string:
    shifted.append(chr(ord(char) + 3))

# Combine back into a string
shifted_string = ''.join(shifted)
print(shifted_string)
```

## Solution

The binary takes user input and compares it against a generated password, which is derived from the string `"sexy1337"` and then obfuscated using a simple XOR with the key `0xAA`. When the user enters the correct password (`vh{|466:`), the program prints the success message `"Access Granted!"`. 

![success](/assets/images/2026-04-11/success.png)
